### Implementation Steps (5 Features)

#### Feature 1: MCP Connection UI (Priority: High ‚Äî Backend ready)
1. **Create `frontend/components/ConnectionsModal.tsx`** ‚Äî Full modal for managing MCP connections
   - List existing connections (GET `/api/mcp/connections`)
   - Add new connection form (name, URL, transport, auth config)
   - Test connection button with live status indicator
   - Delete connection with confirmation
   - Browse tools from connected servers
2. **Update `frontend/components/TopBar.tsx`** ‚Äî Wire "Add custom MCP server" button and connections dropdown to real data
3. **Add `frontend/lib/mcp.ts`** ‚Äî API helpers for all MCP endpoints

#### Feature 2: GitHub Integration (Priority: High)
1. **Create `backend/src/routes/github.ts`** ‚Äî OAuth2 flow + GitHub API proxy
   - `GET /api/github/auth` ‚Äî Redirect to GitHub OAuth authorize URL
   - `GET /api/github/callback` ‚Äî Exchange code for token, store in DB
   - `GET /api/github/repos` ‚Äî List user's repos
   - `POST /api/github/import` ‚Äî Clone a repo's files into sandbox
   - `POST /api/github/push` ‚Äî Push sandbox files to a new branch
   - `POST /api/github/create-pr` ‚Äî Create a pull request
   - `GET /api/github/status` ‚Äî Check if connected
2. **Add `oauth_tokens` table** to schema (provider, access_token, refresh_token, scopes, expires_at)
3. **Create `frontend/components/GitHubModal.tsx`** ‚Äî Connect account, browse repos, import/push
4. **Add env vars**: `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`, `GITHUB_CALLBACK_URL`

#### Feature 3: Google Workspace Integration (Priority: Medium)
1. **Create `backend/src/routes/google.ts`** ‚Äî Google OAuth2 flow
   - `GET /api/google/auth` ‚Äî Redirect to Google consent screen
   - `GET /api/google/callback` ‚Äî Exchange code, store token
   - `GET /api/google/docs` ‚Äî List recent Docs
   - `GET /api/google/sheets` ‚Äî List recent Sheets
   - `POST /api/google/import-doc` ‚Äî Import Doc content as context
   - `POST /api/google/export` ‚Äî Export file to Drive
   - `GET /api/google/status` ‚Äî Check connection status
2. **Create `frontend/components/GoogleModal.tsx`** ‚Äî Connect, browse docs/sheets, import
3. **Add env vars**: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `GOOGLE_CALLBACK_URL`

#### Feature 4: Database Connections (Priority: Medium)
1. **Create `backend/src/routes/databases.ts`** ‚Äî User DB connection management
   - `POST /api/databases/connect` ‚Äî Test and save a connection (Postgres/MySQL/Supabase)
   - `GET /api/databases` ‚Äî List saved connections
   - `DELETE /api/databases/:id` ‚Äî Remove a connection
   - `POST /api/databases/:id/query` ‚Äî Execute a read query (SELECT only)
   - `GET /api/databases/:id/schema` ‚Äî Introspect tables/columns
2. **Add `database_connections` table** to schema (user_id, provider, host, port, database, encrypted_credentials)
3. **Create `frontend/components/DatabaseModal.tsx`** ‚Äî Add/test/browse DB connections

#### Feature 5: Deployment Targets (Priority: Medium)
1. **Create `backend/src/routes/deploy.ts`** ‚Äî One-click deploy orchestration
   - `POST /api/deploy/vercel` ‚Äî Deploy to Vercel (requires token)
   - `POST /api/deploy/netlify` ‚Äî Deploy to Netlify
   - `POST /api/deploy/cloudrun` ‚Äî Deploy to Cloud Run
   - `GET /api/deploy/status/:deployId` ‚Äî Check deploy status
   - `GET /api/deploy/history/:projectId` ‚Äî List past deployments
2. **Add `deployments` table** to schema (project_id, provider, url, status, config, timestamps)
3. **Create `frontend/components/DeployModal.tsx`** ‚Äî Choose target, configure, one-click deploy
4. **Add env vars**: `VERCEL_TOKEN`, `NETLIFY_TOKEN`, `GCP_SERVICE_ACCOUNT_JSON`

#### Shared Infrastructure
1. **Update `backend/src/db/schema.sql`** ‚Äî Add 4 new tables (oauth_tokens, database_connections, deployments, mcp_connections_v2)
2. **Create `backend/src/db/migrations/002_phase2.sql`** ‚Äî Migration file for new tables
3. **Update `backend/src/index.ts`** ‚Äî Register new routers (github, google, databases, deploy)
4. **Update `.env.example`** ‚Äî Add all new env vars with placeholder values
5. **Update `frontend/components/TopBar.tsx`** ‚Äî Live connection status indicators for all integrations
