## Phase 1: Core Functionality Gaps — Implementation Plan

Based on deep codebase exploration, here's the concrete plan to implement all 5 Phase 1 features.

---

### 1. Streaming AI Responses (SSE)

**Backend** (`backend/src/routes/chat.ts`):
- Add new endpoint `POST /api/chat/stream` that sets SSE headers (`text/event-stream`)
- Call Gemini with `chat.sendMessageStream(parts)` and OpenAI with `stream: true`
- Write each chunk as `data: {JSON}\n\n` to the response
- Send `data: [DONE]\n\n` when complete
- Keep the existing non-streaming `/send` endpoint as fallback

**Backend** (`backend/src/services/modelRouter.ts`):
- Add `processChatStream()` that returns an async generator yielding chunks
- Add `callGeminiStream()` using `chat.sendMessageStream(parts)`
- Add `callGPTStream()` using `openai.chat.completions.create({stream: true})`

**Frontend** (`frontend/lib/api.ts`):
- Add `streamMessage()` function using `fetch()` + `ReadableStream` reader
- Parse SSE chunks and yield content tokens

**Frontend** (`frontend/app/(app)/project/[id]/page.tsx`):
- Replace `await sendMessage(...)` with `streamMessage()` call
- Append tokens to the assistant message content incrementally

**Frontend** (`frontend/components/MessageBubble.tsx`):
- No changes needed — it already renders `content` via ReactMarkdown; as content grows, it re-renders

### 2. Real Sandbox Integration (File Tree + Terminal + Live Preview)

**Backend** (`backend/src/services/e2b.ts`):
- Add `listFiles(sandboxId, path)` — calls `sandbox.files.list(path)`
- Add `getPreviewUrl(sandboxId, port)` — calls `sandbox.getHost(port)`

**Backend** (`backend/src/routes/sandbox.ts`):
- Add `GET /api/sandbox/files/:sandboxId` — returns file tree from sandbox
- Add `GET /api/sandbox/preview-url/:sandboxId/:port` — returns the public URL for a port

**Frontend** (`frontend/lib/api.ts`):
- Add `fetchSandboxFiles(token, sandboxId, path?)` 
- Add `executeSandboxCommand(token, sandboxId, command)` — already exists
- Add `fetchPreviewUrl(token, sandboxId, port)`

**Frontend** (`frontend/components/FileTree.tsx`):
- Add `sandboxId` prop, fetch real files via `fetchSandboxFiles()`
- Render actual file structure from E2B
- Add click-to-open (reads file content into CodeEditor)

**Frontend** (`frontend/components/Terminal.tsx`):
- Add command input field at the bottom
- On enter, call `executeSandboxCommand()` and append stdout/stderr to output
- Add ANSI color support (use `ansi-to-react` package)
- Add clear button

**Frontend** (`frontend/components/LivePreview.tsx`):
- Accept dynamic URL from `fetchPreviewUrl()`
- Add refresh button
- Show "No sandbox running" placeholder when no URL

**Frontend** (`frontend/app/(app)/project/[id]/page.tsx`):
- After sandbox starts, poll for files and update FileTree with real data
- After code runs, call `fetchPreviewUrl(sandboxId, 3000)` and update preview

### 3. Code → Sandbox Pipeline

**Frontend** (`frontend/lib/codeParser.ts`) — NEW FILE:
- Parse AI responses for multiple code blocks with file paths
- Regex: ````(\w+)\s*(?:\/\/\s*)?(.+)?\n([\s\S]*?)````
- Return `Array<{ path: string, content: string, language: string }>`
- Detect package.json changes → flag "needs npm install"

**Frontend** (`frontend/app/(app)/project/[id]/page.tsx`):
- After AI responds, parse all code blocks with `parseCodeBlocks()`
- If sandbox is active: write all files via `POST /api/sandbox/write-file`
- If package.json detected: run `npm install` via sandbox command
- Auto-start dev server if not already running
- Refresh file tree and preview URL

**Backend** — no new endpoints needed (write-file and command already exist)

### 4. Persistent File Storage

**Backend** (`backend/src/routes/upload.ts`):
- Replace in-memory `Map` with disk storage (write to `uploads/` directory)
- Store file metadata in DB (new `files` table or add to `projects` metadata)
- On upload: save to `uploads/{projectId}/{fileId}-{filename}`
- `getStoredFile()` reads from disk instead of Map
- Add cleanup on project delete

**Backend** (`backend/src/db/schema.sql`) — add:
```sql
CREATE TABLE uploaded_files (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    project_id UUID REFERENCES projects(id) ON DELETE CASCADE,
    user_id UUID REFERENCES users(id),
    filename VARCHAR(255),
    mime_type VARCHAR(100),
    size_bytes INTEGER,
    storage_path TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Backend** (`backend/src/db/queries.ts`):
- Add `saveFileRecord()`, `getFilesByProject()`, `getFileById()`

**Backend** (`backend/src/db/index.ts`):
- Add in-memory fallback handlers for `uploaded_files` table
