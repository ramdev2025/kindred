### Change 2: Backend — New `src/services/agenticLoop.ts` service

Create a self-contained agentic loop service that:
1. Takes an AI-generated response (code blocks)
2. Parses code files (reuse the same `// filepath:` pattern)
3. Deploys to sandbox via `deployToSandbox()`
4. If deploy fails, constructs a "fix" prompt with the error context
5. Calls the model again (streaming) to get a fix
6. Repeats up to `maxIterations` (default: 3)
7. Yields SSE-compatible events throughout

**Interface:**
```ts
async function* runAgenticLoop(params: {
  sandboxId: string;
  initialCode: string;      // raw AI response with code blocks
  sessionId: string;
  model: string;
  conversationHistory: Message[];
  maxIterations?: number;   // default 3
}): AsyncGenerator<AgenticEvent>
```

**Event types yielded:**
- `{ type: 'iteration_start', iteration: number, maxIterations: number }`
- `{ type: 'deploy_log', content: string }` — from deploy pipeline
- `{ type: 'deploy_result', success: boolean, error?: string, previewUrl?: string }`
- `{ type: 'fix_start', error: string }` — AI is generating a fix
- `{ type: 'token', content: string }` — fix tokens streaming
- `{ type: 'fix_done', content: string }` — full fix response
- `{ type: 'loop_complete', success: boolean, iterations: number, previewUrl?: string }`
