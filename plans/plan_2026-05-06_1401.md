#### Part B: Vercel Blob Storage (Phase 1.4)

**4. Install `@vercel/blob` in backend**
- `cd backend && npm install @vercel/blob`

**5. Create `backend/src/services/blobStorage.ts`** тАФ Storage abstraction
- Uses `@vercel/blob` `put()` / `del()` / `head()`
- Falls back to local disk if `BLOB_READ_WRITE_TOKEN` is not set (dev mode)
- Exports: `uploadToBlob(buffer, filename, mimeType)` тЖТ returns `{ url, pathname }`
- Exports: `deleteFromBlob(url)` тЖТ void
- Exports: `getFileUrl(url)` тЖТ string (for public download links)

**6. Update `backend/src/routes/upload.ts`** тАФ Use blob storage
- After multer processes the file, upload the buffer to Vercel Blob
- Store the blob URL in the response instead of just the local fileId
- Keep local disk as fallback when `BLOB_READ_WRITE_TOKEN` is unset
- Update `getStoredFile()` to fetch from blob URL when the file is stored remotely

**7. Add `project_files` table** тАФ Associate uploads with projects
- Create `backend/src/db/migrations/003_project_files.sql`:
  ```sql
  CREATE TABLE project_files (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id),
    filename VARCHAR(255) NOT NULL,
    mime_type VARCHAR(127) NOT NULL,
    size_bytes INTEGER NOT NULL,
    blob_url TEXT NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
  );
  CREATE INDEX idx_project_files_project ON project_files(project_id);
  ```
- Add DB queries: `saveProjectFile()`, `getProjectFiles(projectId)`, `deleteProjectFile()`

**8. Update upload route** тАФ Store metadata in DB, enforce quotas
- After blob upload, insert into `project_files` with the `projectId` from the request
- Add per-user quota: max 100MB total storage (sum of `size_bytes` across all projects)
- Return quota usage in the upload response
