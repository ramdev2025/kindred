# Frontend Revamp Plan вЂ” Lovable-Inspired UI with Route Groups

## Context

Current state: Basic scaffolded pages with minimal styling. Flat route structure with `/dashboard`, `/project/[id]`. The UI is functional but bare-bones.

**Lovable's UI pattern** uses:
- Clean route group separation: `(root)` for marketing/landing, `(dashboard)` for the authenticated app
- Persistent sidebar navigation in the app shell
- "What do you want to build?" prompt as the central entry point
- Split-panel workspace: chat left, live preview right (no separate editor tabвЂ”code is shown inline in chat)
- Gradient accents, glass-morphism cards, smooth transitions
- Project creation via a modal dialog, not a separate page

## Implementation Steps

### Step 1: Restructure routes with route groups

**Delete/move existing files:**
- `app/page.tsx` в†’ `app/(root)/page.tsx` (landing)
- `app/dashboard/page.tsx` в†’ `app/(dashboard)/page.tsx` (projects home)
- `app/project/[id]/page.tsx` в†’ `app/(dashboard)/project/[id]/page.tsx` (workspace)
- `app/sign-in/` в†’ `app/(root)/sign-in/` 
- `app/sign-up/` в†’ `app/(root)/sign-up/`

**New layout files:**
- `app/(root)/layout.tsx` вЂ” Minimal layout for landing/auth (no sidebar)
- `app/(dashboard)/layout.tsx` вЂ” App shell with sidebar + top bar

### Step 2: Create the Dashboard layout (app shell)

A persistent sidebar with:
- Logo/brand at top
- "New Project" button (opens modal)
- Project list as nav items (scrollable)
- User profile + settings at bottom (UserButton)
- Collapsible on mobile

Top bar with:
- Breadcrumb / current context
- Sandbox status indicator
- Quick actions

### Step 3: Revamp the landing page `(root)/page.tsx`

Lovable-style hero:
- Large gradient headline
- Animated typing effect or prompt input mockup
- Feature cards with icons (glass-morphism style)
- Social proof / "built with" logos strip
- Navbar with Sign In / Get Started

### Step 4: Redesign project workspace `(dashboard)/project/[id]/page.tsx`

Lovable-style split workspace:
- **Left panel**: Enhanced chat with better message rendering (markdown support, syntax highlighting in code blocks)
- **Right panel**: Live preview as default (with tabs for code/terminal/preview)
- A floating "prompt bar" at the bottom center (Lovable's signature)
- File tree sidebar (collapsible, shows sandbox files)
- Status bar at bottom showing model used, tokens, sandbox state

### Step 6: Update `globals.css` with design tokens

- Add custom gradient utilities
- Glass-morphism card styles
- Smooth transition presets
- Custom scrollbar styling
- Animation keyframes for typing indicator

### Step 7: Install additional dependencies

- `lucide-react` вЂ” Icon library (lightweight, tree-shakeable)
- `react-markdown` + `react-syntax-highlighter` вЂ” For rendering AI responses with code blocks
- `framer-motion` вЂ” Smooth animations/transitions

### Step 8: Update middleware

Adjust `middleware.ts` route matcher to reflect the new route group structure (route groups are transparent to URLs, so the middleware paths stay the same вЂ” no change needed).
