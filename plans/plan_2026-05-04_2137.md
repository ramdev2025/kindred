## Key Architecture Decisions
- **Backend**: Express + TypeScript (not full ADK CLI, since that's primarily Python — we'll use the Google Generative AI Node SDK for Gemini access)
- **Auth**: Clerk with `@clerk/backend` for server-side JWT verification
- **Database**: Raw SQL with `pg` (no ORM, keeps it simple)
- **Queue**: BullMQ (modern Bull replacement, Redis-backed)
- **E2B**: Official `@e2b/code-interpreter` SDK
- **Frontend**: Next.js App Router with server components where possible

## Verification Strategy
- Backend: `npm run build` (TypeScript compilation) + manual endpoint testing
- Frontend: `npm run build` (Next.js build) + lint pass
- Service: `python -m pytest` + uvicorn startup check
- Integration: `docker-compose up` brings all services online
