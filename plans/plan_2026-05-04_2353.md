# Plan: File Upload (PDF/CSV/Image) + Google Search Tool

## Context

**Current state:**
- `PromptBar.tsx` ‚Äî text-only textarea, `onSend(message: string)` callback
- `lib/api.ts` ‚Äî `sendMessage()` sends JSON `{ message, projectId, preferredModel }` 
- `backend/src/routes/chat.ts` ‚Äî receives JSON body, passes to `processChat()`
- `backend/src/services/modelRouter.ts` ‚Äî calls Gemini/GPT/Hermes with text-only messages
- Gemini SDK (`@google/generative-ai`) already supports multimodal (inline data parts) and function calling (tools)
- Backend uses `express.json({ limit: '10mb' })` ‚Äî already handles large payloads
- No `multer` or file upload middleware currently installed

**Key insight:** The Gemini SDK natively supports:
1. **Multimodal input** ‚Äî images/PDFs as `inlineData` parts with base64 + mimeType
2. **Google Search as a tool** ‚Äî via `googleSearch` tool in the model config

## Implementation Steps

### Step 1: Install `multer` in backend
Add `multer` + `@types/multer` for multipart form upload handling.

### Step 2: Create upload route ‚Äî `backend/src/routes/upload.ts`
- `POST /api/upload` ‚Äî accepts multipart form with files (PDF, CSV, image)
- Validates file type (pdf, csv, jpg, png) and size (max 10MB)
- Stores files in memory (Buffer) since they're passed directly to AI
- Returns file metadata: `{ fileId, filename, mimeType, size, base64 }`
- Files are stored temporarily in a Map keyed by fileId for the session

### Step 3: Modify `backend/src/routes/chat.ts`
- Add `attachments` field to the request body: `Array<{ fileId: string }>`
- Before calling `processChat()`, resolve attachments to their base64 data
- Pass attachments into the `ChatRequest` interface

### Step 4: Modify `backend/src/services/modelRouter.ts`
- Extend `ChatRequest` interface with `attachments?: Array<{ mimeType: string; data: string; filename: string }>`
- **callGemini**: Use multimodal `parts` array ‚Äî text part + inlineData parts for images/PDFs. For CSV, include the text content directly in the prompt
- **callGPT**: For images, use OpenAI's `image_url` content type. For PDF/CSV, include extracted text in the message
- Add Google Search tool to Gemini calls using `tools: [{ googleSearch: {} }]`

### Step 5: Create `backend/src/services/googleSearch.ts`
- Configure Google Search as a Gemini tool
- When the model decides to search, it returns `functionCall` with search results
- Handle the tool response loop (model requests search ‚Üí execute ‚Üí return results)

### Step 6: Modify frontend `PromptBar.tsx`
- Add a paperclip/attachment button next to the sparkles icon
- Hidden `<input type="file" accept=".pdf,.csv,.jpg,.jpeg,.png" multiple />`
- Show attached file thumbnails/chips below the textarea
- Allow removing attachments before sending
- Change `onSend` signature to `onSend(message: string, files: File[])`

### Step 7: Modify frontend `lib/api.ts`
- Add `uploadFiles(token: string, files: File[])` ‚Äî POSTs to `/api/upload` with FormData
- Modify `sendMessage()` to accept optional `attachments` array of fileIds

### Step 8: Modify workspace page `(app)/project/[id]/page.tsx`
- Update `handleSend` to accept files, upload them first, then send message with attachment IDs

### Step 9: Update `MessageBubble.tsx`
- Show attachment indicators on user messages (file icon + filename chips)
- Show "Searched the web" indicator when assistant response used Google Search

### Step 10: Wire up in `backend/src/index.ts`
- Register the upload route: `app.use('/api/upload', uploadRouter)`
